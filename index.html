Affirm

i worked with the ledger team under Bank Engineering, and had two major tasks:

1. optimizing the ledger health check. i split a previously monolithic task into several child tasks that could run in parallel. i also converted SQL queries into PySpark queries (where applicable). this brought average runtime from ~40 minutes to ~10 minutes.

2. creating the infrastructure for a new tool that would allow engineers outside the team to experiment with the ledger easily. the accounting logic that runs in our daily pipelines is very nuanced, which makes it difficult for other teams to understand how exactly our logic would handle different accounting cases. to help combat this, i helped build a tool that made it easy to run our accounting pipeline ad-hoc/seperate from the full production pipeline. this makes experimenting with the ledger easier, and as a result helps others gain a better understanding of how it works!

Microsoft

worked on a proof of concept that optimized site creation process, from ~15 seconds to ~10 seconds.
the main idea revolved around skipping the wait for DNS registration. instead of waiting we would register a wildcard address that would match the default Microsoft Azure web app domain.
so, after site creation, if a user tried to access the site before the URL was registered with the DNS, their request would be captured by our wildcard address. we could then determine where we should forward their request to.
