Predicting Profanity In YouTube Comments¶

Analysis by Henry Chen and Aranya Banerjee¶

Last updated on 12/15/2018

Introduction¶

The Trending page on YouTube is where YouTube showcases videos it thinks that all sorts of viewers would enjoy. So, the same set of Trending videos is shown to all YouTube users across the country. Typically, trending videos include new music videos from a popular artist or a new movie trailer from a popular franchise. However, there are also more "normal" videos that get their share of the spotlight.

While the videos in the Trending tab are meant for all audiences, there may be certain audiences (such as children and their parents) who would want to avoid videos or comment sections that may be more adult than others. Examining the question of how these audiences may filter this content from their media through the lens of data science provides a case study for the ability of machine learning in protecting consumers from undesired content.

In this project, we aim to use video properties such as title, publish date, genre, and so on to see if we can determine the polarity of a trending video. We will be focusing on Trending videos and their comments in the U.S.

Requirements¶

For the purposes of our project, we will be using a pre-prepared dataset of comments on trending youtube videos from Kaggle, found here. The dataset contains trending videos from 2017.

In addition, we filter for profane words using a list available at FrontGateMedia. note: we've edited the file from FrontGateMedia to get rid of some unnecessary rows and data for easier parsing

We will also be using the following libraries for analysis and visualization:

  • pandas
  • NumPy
  • scikit-learn
  • seaborn
  • matplotlib

Data Modeling¶

We will now try to predict profanity levels based off of various regression and predictive models. We will be using the sklearn library for this.

To start off, let's split the data into training and testing. We will drop columns that are not integers (for the sake of our regression). In different situations and with time allowing, we would explore into how to use NLP in our regression. We believe this would significantly improve result accuracy and usefulness, as it would allow us to factor in things such as tone, genre, and other types of sentiment.

We will split the main dataframe into a 7:3 ratio of testing and training.

In [18]:
train, test = train_test_split(df, test_size = 0.3) # split main dataset into train and test
prediction_var = df.drop(columns=['profanity','video_id','title','channel_title','tags','thumbnail_link', 'percent_upper']).columns
train_X = train[prediction_var] # training data input 
train_y = train.profanity # labeled classifications

test_X = test[prediction_var]
test_y = test.profanity # classification output

Now that we have our training and testing data, let's try some models. First, we will use a linear regressor.

In [19]:
linr = LinearRegression()

linr.score(test_X, test_y)

WOW this hurts. Once again, our score is very small.

If we had more time, we would most definitely find a way to add NLP to our project. This would allow us to analyze things such as titles and tags in a more useful way, such as considering genre, tone, and other sentiment-based analysis.

Closing note: after running several times, it seems the larger our sample, the greater our scores. Due to computational limitations, we capped out at around 50k comments. However, we believe that performance would have improved with more comments (ex: with only a few thousand comments, our scores were negative). That being said, it still would have been not-very-accurate, and we believe being able to use NLP/sentiment analysis on our string-based data would have helped our score significantly.
